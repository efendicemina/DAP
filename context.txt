CONTEXT ZA NASTAVAK RADA - MPR DAP CHATBOT
Datum: 2026-06-03

1. OPSTI OPIS PROJEKTA
- Projekat je NLP/RAG chatbot za pravnu podrsku i digitalno usmjeravanje korisnika Ministarstva pravde BiH.
- Krajnji cilj: korisniku dati tacne, kratke i izvorom potkrijepljene informacije o zakonima, pravilnicima, registrima, obrascima i procedurama, bez davanja pravnog savjeta.
- Ciljni korisnici: gradjani, NVO/udruzenja, fondacije, kandidati za pravosudni/strucni upravni ispit, te svi koji traze administrativne informacije.
- Problem koji rjesava: pronalazenje relevantnih i pouzdanih informacija na mpr.gov.ba kroz prirodni jezik.
- Glavne funkcionalnosti: intent klasifikacija, semanticki retrieval (FAISS + embeddings), reranking sa metapodacima, generisanje odgovora iz izvora, safety i small talk.
- Sistem treba odgovarati na bosanskom/hrvatskom/srpskom jeziku i mora biti source-based (uvek navesti izvore, bez izmisljanja).

5. DATA PIPELINE I SCRAPING
- scraper.py: opsti crawler, radi audit; snima raw zapise i fajlove u chatbot/mpr_dataset_audit/ (all_records.csv, chunks.jsonl, audit_report.html, reports/).
- scraper_v3.py: sitemap-based, filtrira linkove i podrzava dokumente (PDF/DOCX/XLSX); output u chatbot/mpr_dataset_v3_fixed/.
- scraper_v4_curated.py: kurirana lista URL-ova + selected sitemap (selected_sitemap_links.csv), rucni prioriteti; output u chatbot/mpr_dataset_v4/ (records_v4.csv, chunks_v4.csv, coverage_v4.json).
- postprocess.py: stroga filtriranja (URL patterni, GOOD_CATEGORIES, GOOD_TERMS), chunking (MIN 350, MAX 900, OVERLAP 120), deduplikacija; output u chatbot/mpr_dataset_final/.
- update_metadata_v5.py: dodaje page_type, semantic_topic i generic_penalty na v4 chunkove; output u chatbot/mpr_dataset_v5/ (chunks_v5.csv, records_v5.csv, metadata_summary.json).
- Rucno odabrani URL-ovi i sitemap logika postoje u scraper_v4_curated.py; page_type/semantic_topic heuristike u update_metadata_v5.py.

6. INTENT KLASIFIKACIJA
- Dataset generisanje: chatbot/make_intent_dataset.py (TARGET_SIZE=5000, varijacije, tiposi, informalni stilovi).
- Klase/intenti: registracija, obrasci, zakoni_i_propisi, pravna_pomoc, ispiti, registri, kontakt_i_nadleznosti, out_of_scope (+ subintenti u skripti).
- Train/eval: chatbot/train_intent_model.py (TF-IDF 1-3 grams + LogisticRegression class_weight=balanced).
- Evaluacija: chatbot/evaluation.py (accuracy, macro F1, weighted F1, confusion matrix) ali rezultati nisu sacuvani u fajl.
- Model se cuva u chatbot/models/intent_classifier.joblib.

7. RETRIEVAL / RAG SISTEM
- Verzije:
  - TF-IDF baseline: build_rag_index.py (mpr_dataset_final) + test_rag_pipeline.py
  - v2: build_rag_index_v2.py (mpr_dataset_v2)
  - v3: build_rag_index_v3.py (mpr_dataset_v3_fixed)
  - v4: build_rag_index_v4.py (mpr_dataset_v4)
  - Embeddings v4: build_embedding_index_v4.py (models_v4_embeddings)
  - Embeddings v5: build_embedding_index_v5.py (models_v5)
- Embedding model: sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
- FAISS: IndexFlatIP, normalized embeddings (cosine), modeli u chatbot/models_v5/ (faiss_index_v5.bin, chunk_embeddings_v5.npy, embedding_config_v5.joblib)
- Metadata enrichment: page_type, semantic_topic, generic_penalty (update_metadata_v5.py)
- Reranking (chatbot_pipeline_v1.py):
  - +0.20 ako page_type odgovara pitanju
  - +0.25 ako semantic_topic odgovara
  - +0.04 za token match u title
  - -generic_penalty (0.25 za genericke stranice)
  - penalizacija za page_type=ostalo ili semantic_topic=general
- Direktno rutiranje (DIRECT_ROUTE_RULES) za kriticne URL-ove (udruzenja, fondacije, ispiti, pravna pomoc, ministar itd.)
- mMARCO/cross-encoder reranker: Nije pronadjeno u repozitoriju.

Metrike (stvarni brojevi iz rag_eval_results/*.json):
- v4 TF-IDF: Top1=0.5888, Top3=0.6729, Top5=0.7009, MRR=0.6358 (107 uzoraka)
- v4 Embeddings: Top1=0.6355, Top3=0.6822, Top5=0.7477, MRR=0.6718 (107 uzoraka)
- v5 Embeddings: Top1=0.7290, Top3=0.7664, Top5=0.7850, MRR=0.7488 (107 uzoraka)

By-intent (v5, skraceno):
- registracija Top1=0.95 (najjači), obrasci ~0.73, ispiti ~0.54, zakoni_i_propisi 0.40 (slabo), pravna_pomoc ~0.67, kontakt ~0.55.

8. CHATBOT PIPELINE I ANSWER GENERATION
- Glavni fajl: chatbot/chatbot_pipeline_v1.py
- Tok: pitanje -> smalltalk/safety -> intent -> direct route ili embedding retrieval -> reranking -> generisanje odgovora -> odgovor + izvori.
- Answer generation:
  - Primarno: few_shot_composer.compose_few_shot_answer (fact extraction, formatiranje)
  - Fallback: compose_natural_answer (lokalna logika u pipeline)
  - Lokalni LLM preko Ollama: local_llm_answer (USE_LOCAL_LLM=False po defaultu)
- Answer cards: chatbot/answer_cards.py sadrzi strukturisane odgovore (udruzenja, fondacije, takse, ispiti, pravna pomoc). Trenutno nije pozvano u pipeline.
- Sources: vracaju se top 3 izvora (title, url, score, page_type, semantic_topic).

9. SIGURNOST I OGRANICENJA ODGOVORA
- smalltalk detekcija: is_smalltalk() u chatbot_pipeline_v1.py (pozdravi, kako si, itd.)
- harmful detekcija: is_harmful_request() (npr. "kako da otmem", "kako falsifikovati", "kako da prevarim")
- out_of_scope: intent fallback vraca out_of_scope za pasose, licne karte, vozacke, itd; pipeline vraca informativni odbijajuci odgovor.
- Pravilo: chatbot ne smije davati pravni savjet, vec informativno upucivanje na izvore. Ovo je u frontendu istaknuto, a u pipeline-u ukljuceno kroz prompt i fallback.

11. BACKEND API
- FastAPI (backend/main.py). Endpoints:
  - GET / -> HealthResponse (status, message)
  - GET /health -> HealthResponse
  - GET /info -> meta o API + status ML modela
  - GET /suggested-questions -> list[str]
  - POST /chat -> ChatResponse
- POST /chat input: { query: string, session_id?: string, language?: string }
- POST /chat output: { response, confidence, category, sources[], timestamp, query, intent }
- Backend pokusava ucitati chatbot_pipeline_v1.ask(); ako import padne, koristi dummy_responses.
- CORS: allow_origins=["*"] (development).
- session_id je definisan, ali se ne koristi.

Safety je minimalan: smalltalk i osnovna detekcija stetnih zahtjeva. USE_LOCAL_LLM je False; Ollama integracija postoji ali nije aktivna. Za obnovu pipeline-a potrebno je generisati dataset (scraper_v4_curated -> update_metadata_v5), embedding index (build_embedding_index_v5) i intent model (train_intent_model). Najvazniji prvi korak je uskladiti frontend sa backendom i ukloniti dupli UI. Potom treba smanjiti genericke chunkove i poboljsati reranking. Napomena: chatbot_pipeline_v1.py ucitava modele odmah pri importu. Ako neki fajl nedostaje, backend ce prebaciti na dummy_responses.

5. DATA PIPELINE I SCRAPING
Glavni scraping evolucija:
- scraper.py: full crawler + audit; koristi keep/drop sekcije i RELEVANT_TERMS; skida dokumente (pdf/doc/xls), pravi audit izvjestaje u mpr_dataset_audit/.
  - Output: all_records.csv/jsonl, chunks.jsonl, audit_report.html, reports/.
- scraper_v3.py: sitemap-based, fixed, filtriranje linkova po indeksu (MIN_LINK_ID, MAX_LINK_ID). Output: mpr_dataset_v3_fixed/.
- scraper_v4_curated.py: manual URL lista + selected_sitemap_links.csv; jače kuriranje; output u mpr_dataset_v4/ (records_v4.csv, chunks_v4.csv/jsonl, coverage_v4.json).

Ciscenje i chunking:
- postprocess.py: koristi mpr_dataset_audit/all_records.csv; strogi filteri (URL patterns, MIN text 250, GOOD_CATEGORIES/GOOD_TERMS); chunking 350-900 chars, overlap 120; hash dedup.
- Output: mpr_dataset_final/final_kept_records.csv, final_dropped_records.csv, final_chunks.jsonl, qa_dataset_5000_style.jsonl, chat_finetune_dataset.jsonl, final_summary.json.
- Realni brojevi (final_summary.json): original_records 1487, final_kept 99, final_dropped 1388, final_chunks 292, qa_rows 1460, chat_rows 1460.

Metadata v5:
- update_metadata_v5.py: dodaje page_type, semantic_topic, generic_penalty na chunks_v4; output: mpr_dataset_v5/chunks_v5.csv + metadata_summary.json.
- Metadata summary (v5): mnogo genericnog sadrzaja (page_type ostalo=159; semantic_topic general=146) -> potencijalni shum.

6. INTENT KLASIFIKACIJA
- Dataset generator: chatbot/make_intent_dataset.py (TARGET_SIZE 5000, dodaje tipose/informal/context)
- Split: train/val/test (80/10/10)
- Intenti (primarni): registracija, obrasci, zakoni_i_propisi, pravna_pomoc, ispiti, registri, kontakt_i_nadleznosti, out_of_scope.
- Fallback intent: needs_clarification.
- Model: TF-IDF (1-3 grams, min_df=2, max_df=0.95) + LogisticRegression (class_weight balanced).
- Trening: chatbot/train_intent_model.py -> models/intent_classifier.joblib
- Evaluacija: chatbot/evaluation.py i chatbot/test_intent_model.py (accuracy, macro F1, weighted F1, confusion matrix, confidence). Rezultati nisu sacuvani u repo (print u konzolu).

7. RETRIEVAL / RAG SISTEM
Verzije:
- TF-IDF: build_rag_index.py (v1), build_rag_index_v2.py, build_rag_index_v3.py, build_rag_index_v4.py.
- Embeddings: build_embedding_index_v4.py, build_embedding_index_v5.py.

Aktivni pipeline (chatbot_pipeline_v1.py):
- Embedding model: sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
- FAISS: IndexFlatIP sa normalized embeddings (cosine-like)
- Ulazni podaci: mpr_dataset_v5/chunks_v5.csv
- Artefakti: models_v5/faiss_index_v5.bin, chunk_embeddings_v5.npy, embedding_config_v5.joblib

Retrieval logika:
- intent prediction + keyword fallback + contact override
- direct_route_rules (hardkodirane rute ka konkretnim URL-ovima)
- embedding_candidates -> rerank_score (page_type match, semantic_topic match, title token boost, generic_penalty)
- Nema cross-encoder ili mMARCO reranker (nije pronadjeno u repozitoriju)

8. CHATBOT PIPELINE I ANSWER GENERATION
Glavni fajl: chatbot/chatbot_pipeline_v1.py
Tok:
1) Normalizacija pitanja
2) Smalltalk/harmful detekcija
3) Intent klasifikacija (+ fallback)
4) Retrieval (FAISS + reranking + direct routes)
5) Answer generation (LLM opciono) + few-shot composer
6) Izlaz: odgovor + izvori

Answer generation:
- USE_LOCAL_LLM = False po defaultu; lokalni LLM podrzan preko Ollama (OLLAMA_URL http://localhost:11434/api/generate, OLLAMA_MODEL qwen2.5:3b).
- Primarni put bez LLM: few_shot_composer.compose_few_shot_answer (goal detection: procedure/requirements/fee/contact/form/definition/date/general; extraction facts; formatiranje odgovora)
- Fallback: compose_natural_answer (jednostavniji, vise raw facts)
- sources: top 3 jedinstvena URL-a sa title/url/score/page_type/semantic_topic.

Answer cards:
- chatbot/answer_cards.py ima hardkodirane kartice (osnivanje udruzenja, dokumentacija, taksa, promjena, fondacije, ispiti, besplatna pravna pomoc, medjunarodna otmica djece).
- U pipeline se importuje match_answer_card, ali se NE koristi (mrtav kod / nedovrseno).

9. SIGURNOST I OGRANICENJA ODGOVORA

18. KRATKI SAZETAK ZA NASTAVAK RADA
Projekat je NLP/RAG chatbot za Ministarstvo pravde BiH koji usmjerava korisnike na sluzbene izvore i ne daje pravni savjet. Backend je FastAPI sa /chat endpointom i pokusava da koristi chatbot_pipeline_v1, uz fallback na dummy_responses ako ML modeli ne ucitaju. Glavni pipeline koristi v5 embeddings (SentenceTransformer + FAISS) i metapodatke page_type/semantic_topic za reranking. Intent model je TF-IDF + LogisticRegression i nalazi se u chatbot/models/intent_classifier.joblib. Dataset je agresivno filtriran: 99 kept od 1487, 292 chunkova, 1460 QA parova. Scraping ima vise verzija, a v4_curated koristi manual URL listu i sitemap; metadata v5 dodaje page_type, semantic_topic i generic_penalty. Retrieval evaluacija (107 pitanja) pokazuje da v5 embeddings imaju top1 0.729 i mrr 0.7488, ali zakoni_i_propisi i ispiti su slabi po intentu. U pipeline postoji optional Ollama LLM, ali je USE_LOCAL_LLM=false po defaultu. Frontend je Next.js 14, ali ima dva UI-ja: aktivni app/page.tsx ne prikazuje izvore ni confidence, dok components/ChatInterface prikazuje izvore i confidence. Confidence prikaz je ranije uklonjen zbog konfuzije, pa treba paziti koja komponenta se koristi. Answer_cards postoje, ali nisu ukljucene u pipeline (mrtav kod). Safety logika je minimalna (smalltalk + par harmful fraza). .env.example nije prisutan, a konfiguracija API URL-a ide preko NEXT_PUBLIC_API_URL. Repo sadrzi velike dataset/model direktorije i root .gitignore je nedovoljan; treba prosiriti ignore ili LFS. Najhitnije je konsolidovati frontend i izvore, zatim pojacati metadata filtering i reranking, i dodati evaluaciju kvaliteta finalnog odgovora. Ovo ce dati najveci skok u kvalitetu bez promjene osnovne arhitekture.
